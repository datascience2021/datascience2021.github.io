Installing Active Directory and configuring Group Policies in Windows Server 2016.


Step 1:    Before starting the Windows Server 2016 virtual machine make sure you have at least 4GB RAM. And confirm that the network adapter is configured for NAT. The Windows 10 VM should have 3GB RAM and NAT as well. But do not start the W10 VM yet.    

Step 2:    Start the Windows Server 2016 VM, log in as Administrator using Pa$$w0rd and make sure that it has these parameters:
  • IP address = 192.168.1.200
  • Subnet mask = 255.255.255.0
  • DNS IP address = 192.168.1.200

Step 3:    Install Active Directory using Server Manager. Since the way to do that is similar to Window Server 2012 you may want to take a look at this link to remember the steps. The AD domain is krebner.local.

Step 4:    Start the Windows 10 VM and after configuring the network paramenters (IP = 192.168.1.20, SM = 255.255.255.0 and DNS = 192.168.1.200), join the krebner.local domain. Remember you can join the domain by typing sysdm.cpl from the command line and then clicking the change button.

Step 5:    Before logging in to the Windows 10 VM, create three organizational units (using Active Directory Users and Computers from the Server Manager) in Active Directory: Marketing, Sales and Accounting. Add these users to the OUs (one on each): Emmanuel Macron, Angela Merkel and Theresa May.

Step 6:    On the Windows 10 VM log in as the three users you've just created in AD. Remember that this step takes a while. After doing that, log off. We'll use those accounts to test group policies (GPOs).

Step 7:    Using the Group Policy Management tool (on Server Manager) create a Starter GPO named MKT_Starter as seen on the pictures below. The two settings are for preventing access to the command prompt and to the registry.

Step 8:    Now create a new GPO named MKT_GPO using the starter GPO. And link the GPO to the Marketing OU. Close the GPMC (Group Policy Management Console).

Step 9:    Now on the W2016 VM type gpupdate /force from the command line. To practice the PS (PowerShell), type the Invoke-GPUpdate commandlet too.

Step 10:    Log in to the W10 VM as Emmanuel Macron (from the Marketing OU) and verify the command line and registry edition restrictions. Do that by trying to access the command prompt and trying to run the regedit utility. Did the GPO work? After the verification log off.
